利用Github和Hexo搭建个人博客

发表于 2020-01-25

这篇文章简略概括利用Github和Hexo搭建个人博客的步骤。Github是一个面向开源及私有软件项目的托管平台，本身可用来下载和学习大神们的代码，其中的Repositories(仓库)可用来储存个人博客的数据。Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown(或其他渲染引擎)解析文章，可利用靓丽的主题生成静态网页。关于如何利用这两样工具来搭建博客，网上已有很多的教程。本网站主要参考Yankun's Blog和小茗同学的博客园而成功搭建。现在只简介搭建的主要步骤，至于其中的细节可参照上述两篇博文。

·前奏

Github

首先注册Github账号，在自己的Repositories下新建一个仓库。需要注意新仓库名必须以username.github.io格式命名，且一个账号只能创建一个该类型仓库。其中username 是自己Github的用户名，比如我的仓库名是MingLi-phy.github.io。

安装Hexo

参考Hexo官方教程，对于win用户(以下内容只适用于window系统，其他系统安装Hexo的过程请参考Hexo官方教程)，主要是先安装两个应用程序Git和Node.js(10.0 及以上版本)。

Git配置初始化：上述的两个应用程序安装完成后，在桌面点击鼠标右键，选择Git Bash Here，打开后输入：

git config –global user.name “username”

git config –global user.email “youremail@example.com“

其中，”username” 和 “youremail@example.com“ 分别是自己的Github用户名及注册Github时用的邮箱。

配置完成后，用以下代码查看自己的配置信息:

git config –list

连接Git和Github：鼠标右键任意地方，选择 Git Bash Here ， 输入：

ssh-keygen -t rsa -C “youremail@example.com“

回车运行后，最终会在C:\Users\Administrator生成一个.ssh文件，打开后会发现文件id_rsa.pub。然后右键用文本编辑器打开，用 ctrl + A 全选然后ctrl + C 复制。

接着进入Github主页，选择setting，创建新的 SSH key ，其中title随意，在key里面粘贴刚才复制的内容，点击 Add SSH key 完成。

使用 npm 安装 Hexo：鼠标右键任意地方，选择 Git Bash Here ， 输入：

npm install -g hexo-cli

没有出现error，即为安装成功。

·间奏

利用Hexo建立博客

在磁盘下新建一个空文件夹。接下来关于博客的所有内容都将在此文件夹下进行。在这个空文件夹内右键打开 Git Bash Here ，输入：

hexo init

以上命令建立了网站所需的初始文件，接着安装依赖包：

npm install

然后配置本地文件到Github

npm install hexo-deployer-git –save

接着用文本编辑器 (推荐Sublime Text3) 打开_config.yml文档，在文档底部填写depoly信息。

deploy:
  type: git
  repository: https://github.com/username/username.github.io.git  
  #username处替换成自己的用户名
  branch: master

预实验

通过上述一系列操作，其实已经建立了最简单的博客。现在可以预览一下效果了。在博客文件所在的文件夹下右键Git Bash Here，然后输入：

hexo g

hexo s

先不关闭git页面，打开浏览器，输入网址http://localhost:4000即可本地查看自己的博客啦！注意这里只是本地查看，别人是看不到的。这个操作过程同样适用于以后发博客前的预览草稿。

更换主题

Hexo默认的主题是Landscape，这也是刚才预览时所看到样子。Hexo官网提供了不同风格的博客主题，本网站所采用的是Ayer Theme。

更换主题的方式：首先在官网主题中选出自己喜欢的风格，这应该是所有步骤中最花时间的了。然后进入该作者的Github账号下下载压缩文件到本地电脑里。接着将其解压到博客所在文件夹的themes文件夹中，注意解压后文件的名字。最后配置Hexo的_config.yml文档，用文本编辑器打开该文档，找到theme一行，将landscape替换为刚才解压文件的名字。接下来就可以预览一下效果。

不同的主题自带有其专属的配置文件，例如Ayer主题本身也自带有一个_config.yml文档，可根据作者的提示来调整博客的外观。

书写文章

回到博客的根目录下 (即拥有public文件的那个文件夹)，右键Git Bash Here执行：

hexo new ‘my-first-blog’

然后会发现在source/_posts文件夹下生成了一个名为my-first-blog.md的文档。其中后缀md代表该文档可按照Markdown语言编辑。因此以后就按此格式书写文章，所以需要了解一些Markdown的基本语法。

写好文章后，回到博客的根目录下，可通过以下命令将其更新到网站上：

hexo g

hexo d

然后登陆网址username.github.io就可以查看自己发布的博客啦。
