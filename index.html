漫话物理史(一)(古典物理学)

2020-01-27

昨日繁星点月当空，辉光所至遍起楼层。今朝廊桥亭台水榭，不见当时璀璨虹星。

  当今理论物理学已经很久没有突破性进展了，这并不是说所有的谜题都已被解开，而是反衬出先贤们建立的理论框架太过辉煌。从物理的旷野上望去，两栋摩天大楼最为瞩目，它们的楼盘上分别标注着广义相对论和量子场论，然而它们并没有连接在一起，而像是支撑天空的两根独立的柱子。抬头仰望天空，到处漂浮着积云。淡薄的云层在振荡 (中微子振荡)，暗处的密云在滋长 (暗物质与暗能量)，仿佛要招致一场暴雨，将高楼完全冲塌，让未来流失在未知的方向。本系列博文将要回顾物理学的发展历史，跟随巨星们的脚步感受理性的光芒，眼看着他们如何地起高楼。不得不说，那将是一个很长很长的故事……

一、古典物理学

1. 仰望星空的泰勒斯

  在两千多年前 (约公元前6世纪) 某天的夜晚，泰勒斯走在古希腊某田野的泥路上。他一边望着遥远的星空，一边自说自话若有所思，不出所料，他摔进了一个深坑 (话说为什么那时候路面会被刨个坑呢？→_→)，他浑身伤痕累累，难以从坑中爬出。幸好有一个商人路过将他救起。泰勒斯为表示感谢，告诉了商人他的伟大发现，“明天会下雨！”，泰勒斯兴奋地说道。商人本以为会得到很好的报酬，此时却大失所望。但商人还是不打算亏本，他准备将这个事情当作笑料传遍足迹所到之处。从此一句名言流传开来：哲学家是只知道天上的事情而不知道脚下发生什么事情的人。多亏好事商人的努力，泰勒斯成为了人尽皆知的仰望星空的明星。如今人们追溯物理学的起源时，往往把泰勒斯仰望星空作为标志性事件，从此观测与预言这样的组合成为了理性研究的范式。

  泰勒斯的故事还没有完结。他将目光投向了爱琴海的东岸，那里已持续了近三十年的战争。他准备用知识来停止这场灾难。泰勒斯自告奋勇当说客分别到两阵营主将那里谈判，“你们愚蠢的斗争已引起神的震怒，他将要让太阳失去光辉。” 双方将领都不以为然。待到再一天两方摆开阵型时，日食如期而至，士兵们恐慌万分，纷纷逃窜。主将们这才坐在一起签订合约永不再战。这也是第一个史料记载的被预测的日食。

2. 全才的亚里士多德

  在公元前四世纪的某天，一位古希腊的公子正在柏拉图学院与老师激烈争论。他几乎不同意老师的各种观点。这位老师即是创办学院的柏拉图。柏拉图师从苏格拉底，继承着苏格拉底的智慧，并著有理想国一书来阐述自己的哲学理念。在柏拉图的思想中，他认为人的意识中存在着一个完美的国度，世间万物不过是意识的不完美的投射。比如，当我们眼睛看到一匹马，柏拉图会这样解释到，人的意识中早就存在一个完美的马的意象，而现在的这匹马满身尘土，体格薄弱，是意识向现实妥协的一个有缺陷的映射，当我们看到现实的马时，完美的马的意象就会被唤醒，从而由眼睛告诉大脑，哈，那是一匹马。亚里士多德却觉得马本来就存在，它就是独一无二的它，为什么要弯弯绕绕再引入一个看不见摸不着的更完美的它呢？简单点，说话的方式简单点。师徒二人僵持不下，亚里士多德放出豪言：吾爱吾师，吾更爱真理！

  于是他打算重新思索学习的过程：人们到底是怎么获得关于外物的知识的呢？终于他迈出了认知外物的第一步：分类。将世间万物按一定规律分类为不同的集合。每个集合内部的事物拥有共同的性质，而集合之间的事物要有明显的不同。然后将每个集合起一个新的名字来定义它们。学习体现在哪里呢？这时将事物分类的过程就是学习。分类时由于需要知道所定义集合的共性，这就需要通过感官逻辑等主观要素来掌握事物的不同特征。分类需要有一个前提，那就是第一个分类事物的人必须博学多闻，能建立一个內恰的框架。幸运的是，亚里士多德就是那个全才博学的人。他几乎对每个学科都做出了贡献。他的写作涉及伦理学、形而上学、心理学、经济学、神学、政治学、修辞学、自然科学、教育学、诗歌、风俗，以及雅典法律。亚里士多德的著作构建了西方哲学的第一个广泛系统，包含道德、美学、逻辑和科学、政治和玄学。有趣的是，在机器学习/人工智能发展的如火如荼的两千年后的今天，电脑深度学习的头等要事也是分类，一个能精准执行分类任务的AI才是会学习的好AI。

  伟大的人物也会被时代所限，事实上亚里士多德的分类标准过于简单，而一些结论也下的比较草率，这也导致了那时人们的思想长期受到束缚，无法思考更为深刻和广阔的空间。人的命运啊，当然要靠自我奋斗，但也要考虑历史的进程。因此我们评价一位历史人物时要代入到他所在的时代来评价高低。毫无疑问，在古希腊时代，希腊三贤的功绩无出其右。

3. 阿基米德与浮力定理

  时间来到公元前三世纪的一天夜晚，阿基米德正躺在澡盆里苦苦思索。原来古希腊的希洛国王命令他检测一个金冠是否掺假。他知道金属是否掺假可通过测量它们的密度来判断。如果金冠掺了劣质铝铁，那么密度就会减小。而如果想知道一个物体的密度，那首先要测得它的质量和体积。对于质量呢，可以用杆秤来称量。但是怎么测一个不规则体形的金冠的体积呢？阿基米得看着澡盆里的水面起起伏伏，突然脑袋旁边亮起一个”灯泡”：人本身也是一个不规则物体啊，我进入澡盆里水面上升，不是因为水被我的体积排开了吗？于是第二天他向国王借来与金冠等质量的真金子。他的逻辑线已逐渐清晰起来：质量一定，密度小的物体体积大！将金冠和等质量的真金分别投入水中，如果金冠掺假了那么它会导致水面上升更高！制作王冠的工匠们后来的命运已无史料记载，但阿基米得解决问题的方式却以另外一种方式流传下来，那就是浮力定理(浸入静止流体中的物体会受到一个浮力，其大小等于该物体所排开的流体重量，方向沿流体法向并通过所排开流体的形心)。在中国古代的三国时期，曹冲利用了与阿基米得同样的方法测得了大象的体重。

  阿基米得有一句名言：给我一个支点，我可以撬动地球！这当然是一种夸张的说法，因为根据由他自己提出的杠杆平衡原理推知，如果阿基米德想凭借自身重力撬起地球的话，即使一切实验设备都准备好了，而且他能够以光速运动，他也需要6万年的时间才能将地球撬起一厘米。虽然实验上无法为他验证，但是理论上的可能性从未被人们怀疑。

利用Github和Hexo搭建个人博客

2020-01-25

这篇文章简略概括利用Github和Hexo搭建个人博客的步骤。Github是一个面向开源及私有软件项目的托管平台，本身可用来下载和学习大神们的代码，其中的Repositories(仓库)可用来储存个人博客的数据。Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown(或其他渲染引擎)解析文章，可利用靓丽的主题生成静态网页。关于如何利用这两样工具来搭建博客，网上已有很多的教程。本网站主要参考Yankun’s Blog和小茗同学的博客园而成功搭建。现在只简介搭建的主要步骤，至于其中的细节可参照上述两篇博文。

·前奏

Github

首先注册Github账号，在自己的Repositories下新建一个仓库。需要注意新仓库名必须以username.github.io格式命名，且一个账号只能创建一个该类型仓库。其中username 是自己Github的用户名，比如我的仓库名是MingLi-phy.github.io。

安装Hexo

参考Hexo官方教程，对于win用户(以下内容只适用于window系统，其他系统安装Hexo的过程请参考Hexo官方教程)，主要是先安装两个应用程序Git和Node.js(10.0 及以上版本)。

Git配置初始化：上述的两个应用程序安装完成后，在桌面点击鼠标右键，选择Git Bash Here，打开后输入：

git config –global user.name “username”

git config –global user.email “youremail@example.com“

其中，”username” 和 “youremail@example.com“ 分别是自己的Github用户名及注册Github时用的邮箱。

配置完成后，用以下代码查看自己的配置信息:

git config –list

连接Git和Github：鼠标右键任意地方，选择 Git Bash Here ， 输入：

ssh-keygen -t rsa -C “youremail@example.com“

回车运行后，最终会在C:\Users\Administrator生成一个.ssh文件，打开后会发现文件id_rsa.pub。然后右键用文本编辑器打开，用 ctrl + A 全选然后ctrl + C 复制。

接着进入Github主页，选择setting，创建新的 SSH key ，其中title随意，在key里面粘贴刚才复制的内容，点击 Add SSH key 完成。

使用 npm 安装 Hexo：鼠标右键任意地方，选择 Git Bash Here ， 输入：

npm install -g hexo-cli

没有出现error，即为安装成功。

·间奏

利用Hexo建立博客

在磁盘下新建一个空文件夹。接下来关于博客的所有内容都将在此文件夹下进行。在这个空文件夹内右键打开 Git Bash Here ，输入：

hexo init

以上命令建立了网站所需的初始文件，接着安装依赖包：

npm install

然后配置本地文件到Github

npm install hexo-deployer-git –save

接着用文本编辑器 (推荐Sublime Text3) 打开_config.yml文档，在文档底部填写depoly信息。

deploy:
  type: git
  repository: https://github.com/username/username.github.io.git  
  #username处替换成自己的用户名
  branch: master

预实验

通过上述一系列操作，其实已经建立了最简单的博客。现在可以预览一下效果了。在博客文件所在的文件夹下右键Git Bash Here，然后输入：

hexo g

hexo s

先不关闭git页面，打开浏览器，输入网址http://localhost:4000即可本地查看自己的博客啦！注意这里只是本地查看，别人是看不到的。这个操作过程同样适用于以后发博客前的预览草稿。

更换主题

Hexo默认的主题是Landscape，这也是刚才预览时所看到样子。Hexo官网提供了不同风格的博客主题，本网站所采用的是Ayer Theme。

更换主题的方式：首先在官网主题中选出自己喜欢的风格，这应该是所有步骤中最花时间的了。然后进入该作者的Github账号下下载压缩文件到本地电脑里。接着将其解压到博客所在文件夹的themes文件夹中，注意解压后文件的名字。最后配置Hexo的_config.yml文档，用文本编辑器打开该文档，找到theme一行，将landscape替换为刚才解压文件的名字。接下来就可以预览一下效果。

不同的主题自带有其专属的配置文件，例如Ayer主题本身也自带有一个_config.yml文档，可根据作者的提示来调整博客的外观。

书写文章

回到博客的根目录下 (即拥有public文件的那个文件夹)，右键Git Bash Here执行：

hexo new ‘my-first-blog’

然后会发现在source/_posts文件夹下生成了一个名为my-first-blog.md的文档。其中后缀md代表该文档可按照Markdown语言编辑。因此以后就按此格式书写文章，所以需要了解一些Markdown的基本语法。

写好文章后，回到博客的根目录下，可通过以下命令将其更新到网站上：

hexo g

hexo d

然后登陆网址username.github.io就可以查看自己发布的博客啦。

第一篇博客

2020-01-25

  一直以来都想找到一个写字的地方，记录下平凡的一天。以前曾尝试过用Onenote, Goodnotes，Latex等记录课堂笔记，这些主流工具确实还不错。但Onenote和Goodnotes不能友好的支持数学公式，而Latex虽然能够满足日常所有的格式要求 (毕竟是用来写论文的) 但语法稍繁琐。另一方面有道云笔记虽支持语法简洁的Markdown但平时并不常使用。在某天搜索贝叶斯原理时遇见了一篇优秀的博文Yankun’s Blog，在拜读完其机器学习的文章后，又迫不及待地点开了搭建博客的博文，毫无疑问那是一篇高质量的作品，细节介绍的非常清楚。我便照葫芦画瓢也实践了起来。

  不同于课堂笔记，这里主要记录的是文字和心情。又因为没有绑定域名，所以就形成了个人的专属空间。在博客的主题上，这里选择了Ayer。Ayer在马来语中是“水”的意思，在西班牙语中是“昨天”的意思，有趣的寓意于是在主页没有更换。网站内拥有五个标签目录，是对不同风格文字的初分类。

  在日常标签下，主要是记录平时有趣的经历和想法。名为日常，但何时更新却要随缘啦 (某症快来背锅→_→)。在随笔标签下，会写一些短文来记录情绪和思考。与自己对话，于是用开导性语言。在收藏标签下，主要是收集网上优质的资源，并进行应用、总结和评价。如同一个书柜，里面的藏书以后可能会去翻阅，也可能永成装点。在集思标签下，会提出一些与工作和学习有关的想法，特点是游思妄想。在心流标签下，希望能够专注地做一些事情。心流(Mental flow) 在心理学中是指一种人们在专注进行某行为时所表现的心理状态，是一种将个人精神力完全投入在某种活动上的感觉。心流产生的同时会有高度的兴奋及充实感。在TED上某一期有位大叔提出一个有趣的观点：为什么我们做事情总是虎头蛇尾？那是因为我们对那件事只有兴趣！兴趣是吸引是开始，但不构成动机(Motivation)。因此，对于真正喜欢的事需要有Passion：你能为其奉献到何种地步？忘我即是心流。
